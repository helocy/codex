## 一、产品概述

### 1.1 产品定位

Codex 是一个**本地优先**的智能知识库管理系统，旨在成为用户的"第二大脑"。系统将个人文档（Markdown、PDF、Word、纯文本）转化为可检索、可对话的结构化知识库，通过混合检索技术和 RAG（检索增强生成）与大模型深度结合，让用户能够用自然语言查询自己积累的知识。

### 1.2 核心价值主张

| 特性 | 说明 |
|------|------|
| **本地部署** | 所有数据存储在本地，无需上传到第三方云服务，隐私安全 |
| **知识沉淀** | 上传文档自动切块、向量化，形成可检索的个人知识库 |
| **混合检索** | BM25 关键词 + 向量语义双路融合，技术术语和自然语言都能精准匹配 |
| **RAG 对话** | 基于知识库上下文与大模型多轮对话，回答有依据、有来源 |
| **多模型支持** | 兼容豆包、通义千问、OpenAI、Ollama 等任意 OpenAI 兼容接口 |
| **一键部署** | 单脚本完成全套环境安装，其他电脑 5-15 分钟内即可运行 |

### 1.3 目标用户

- 有大量 Markdown 笔记/技术文档的开发者和研究人员
- 希望用 AI 检索、问答自己知识库的知识工作者
- 对数据隐私有要求、不愿使用云端知识库产品的用户

## 四、核心算法

### 4.1 Markdown 感知分块

普通字符切割会破坏文档的语义单元。Codex 使用三层策略：

```
文档
 ├── 按 #/##/### 标题切分 section
 │    ├── section ≤ 1000字符 → 直接作为一个 chunk（保留标题）
 │    └── section > 1000字符 → 按 \n\n 段落合并
 │         ├── 合并后 ≤ 1000字符 → 一个 chunk
 │         └── 单段落超限 → 字符切割（overlap=100字符）
 └── 每个 chunk 携带所属标题作为上下文前缀
```

### 4.2 混合检索 + RRF

```
用户查询
   ├─→ 向量检索（sentence-transformers 余弦相似度排序）
   └─→ BM25 关键词检索（rank_bm25，\w+ 分词）
                          ↓
               RRF(k=60) 融合两路排名
               score = Σ 1/(60 + rank_i)
                          ↓
               相邻 Chunk 合并（同文档连续索引）
                          ↓
               返回 top-K 结果
```

**BM25 的价值：** 向量模型对语义相近的表达理解好，但对专有名词（芯片型号、API 函数名、错误码）容易遗漏，BM25 精确匹配弥补了这一盲点。

### 4.3 PageIndex 两阶段检索

受 [VectifyAI/PageIndex](https://github.com/VectifyAI/PageIndex) 启发实现的树形导航检索：

```
用户查询
   │
   ▼ 阶段1：树形导航（LLM）
   BM25 预筛文档标题 → Top-15 相关文档
   ↓
   将各文档压缩树摘要（title + summary[:100]，最多 3 层）送给 LLM
   LLM 输出相关 node_id 列表（最多 15 个节点）
   │
   ▼ 阶段2：精细检索
   按 section_id 过滤 chunks（仅目标章节）
   ↓
   向量 + BM25 + RRF 混合检索（取 top_k/2）
   ↓
   全量检索兜底（取 top_k 补充未覆盖的 chunk，防止阶段1选错节点）
   ↓
   返回 top-K 结果 + 命中节点列表
```

**树形索引生成策略：**
- Markdown：直接解析 `#/##/###` 标题层级，无需 LLM
- PDF：按 30 页批次调用 LLM，提取页码与章节层级
- 文本/Word：LLM 按逻辑段落划分，均匀分配行号范围

**tree_index 节点结构：**
```json
{
  "node_id": "0042",
  "title": "快速启动配置",
  "start_index": 50,
  "end_index": 65,
  "summary": "介绍 RV1126B 平台的快启 BoardConfig 配置方法",
  "nodes": [...]
}
```

### 4.4 RAG Prompt 策略

```
system: 规则指令（只用文档内容，找不到说找不到，可引用片段编号）
history[-12]: 最近 6 轮对话（保持上下文连贯）
user:   [参考文档片段 1..N]
        [当前问题]
```

## 十、常见问题

**Q: 首次启动很慢？**
A: sentence-transformers 首次使用时自动下载模型（~500MB），之后从本地缓存加载。

**Q: 支持哪些大模型？**
A: 任何兼容 OpenAI Chat Completions API 的服务。在「设置」页填入 Base URL、API Key、模型名即可。

**Q: 知识库问答效果不好？**
A: 先去「管理」页重置知识库，再重新上传文档（新分块策略更完整）。确认「对话」页已开启「使用知识库」开关。

**Q: 如何迁移到新电脑？**
A: 新电脑执行 `bash deploy.sh`，然后用 `pg_dump` / `pg_restore` 迁移数据库，将 `backend/.env` 和 `uploads/` 目录复制过去。
